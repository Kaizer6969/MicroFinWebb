what is lookback?

Thought for a few seconds
Lookback means the past time window the system checks before deciding.

Example:

Late payments ≥ 2 within 6 months
→ the 6 months is the lookback period

No defaults within 12 months
→ the system checks the borrower’s records from the last 12 months

So in simple terms:

rule = what to check

lookback = how far back to check it

Examples
Successful repayment cycles ≥ 3 within 12 months

No overdue within 6 months

Default count = 0 within 24 months

Late payments ≥ 2 within 3 months

Why it matters
Without lookback, the rule becomes unclear.

For example:

No defaults ever = checks all history

No defaults within 12 months = checks only recent history

That’s why “default history” felt vague earlier.
It should usually be written as either:

no defaults ever

no defaults within X period

default count within X period

For your UI, you can think of it as:

Condition

Threshold

Lookback period

Example:

Condition: Late payments

Threshold: 2 or more

Lookback: 6 months

So yes, lookback = review period. “Review period” might even be a friendlier label in the UI.

A. Fully automatic
If eligibility passes, system upgrades/downgrades automatically.

Good:

fast

consistent

easy for tenants who want automation

Bad:

B. Recommend only

Good:

safer

more realistic for financial institutions

better control

Bad:

slower

adds manual workload

C. Hybrid
Upgrade can be automatic or approval-based

Downgrade usually creates a review case first, unless severe triggers happen

This is the model I’d recommend first.

My recommended default:
Upgrade action = optional auto-upgrade or admin approval

Downgrade action = default to manual review, except critical triggers
